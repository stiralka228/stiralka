В рамках лабораторной работы необходимо настроить процесс непрерывной интеграции (CI) с использованием GitHub Actions для выбранного проекта (Вариант выбирается согласно порядковому номеру в ЭУ. Проекты не должны повторяться.).

Основная цель — автоматизировать проверку качества кода при каждом изменении в репозитории.

Для этого требуется:
- Выбрать существующий публичный репозиторий или использовать собственный проект
- Убедиться, что в проекте отсутствуют готовые CI-конфигурации
- Добавить тесты (если их нет), покрыв минимум 3 функции или метода
- Создать workflow-файл .github/workflows/ci.yml

Pipeline должен включать три обязательных этапа:
1. Lint (статический анализ кода). Проверка кода на стиль, потенциальные ошибки и несоответствие стандартам
2. Test (запуск тестов). Автоматический запуск тестов для проверки корректности работы программы. Желательно использовать покрытие кода (coverage)
3. Build (сборка проекта). Создание итогового артефакта (пакета, бинарника, Docker-образа и т.д.). 

После настройки необходимо убедиться, что workflow:
1. автоматически запускается при push в ветку main
2. успешно проходит все этапы
